# components/project/sims/

## Purpose

Client-side simulators for netbox-sdk demo commands. Every output string they
display must originate from a committed fixture written by
`scripts/sync-netbox-sdk-fixtures.ts` from the local `netbox-sdk` repo.

## Files

- `useFixture.ts` - Fetches `/netbox-sdk-fixtures/<name>` with an in-module cache and in-flight de-dupe map.
- `DemoInitTrace.tsx` - Replays the `nbx init` prompt flow from `demo-init-flow.json`. Typed answers are simulated; prompt labels and success lines are fixture-derived.
- `DemoDevicesList.tsx` - Streams captured stdout from `demo-devices-list.json`, generated by running `nbx demo dcim devices list` in netbox-sdk and rewriting displayed argv to omit `demo`.
- `DemoTuiModal.tsx` - Full-screen TUI simulation dialog backed by `tui-simulation/main-browser.json` and generated SVG state captures. Hotspots come from the fixture manifest.

## Fixture Contract

Fixtures live under `public/netbox-sdk-fixtures/`:

- `demo-init-help.json`, `demo-tui-help.json` - docgen capture JSON from netbox-sdk.
- `demo-devices-list.json` - live CLI capture from the local netbox-sdk checkout.
- `demo-init-flow.json` - prompt labels and success line extracted from `netbox_cli/demo.py`.
- `netbox-sdk-metadata.json` - release, Python lower bound, and supported NetBox versions derived from netbox-sdk.
- `tui-main-*.svg` and `tui-simulation/*` - generated TUI screenshots and interactive state manifest.
- `manifest.json` - source repo, commit, timestamp, and fixture list.

## Adding A New Sim

1. Add or identify the netbox-sdk source artifact.
2. Extend `scripts/sync-netbox-sdk-fixtures.ts`.
3. Add the sim component that consumes data through `useFixture()`.
4. Wire the runner into the project page surface.

## Hard Rule

No file in this folder may contain terminal output as literal constants. Grep
for `nbx `, `$ `, `|`, box drawing characters, and table fragments before
reviewing changes.
