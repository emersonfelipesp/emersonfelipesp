# public/

## Purpose

Static assets served directly by Next.js at the root URL path. Files here are
publicly accessible and bypass the normal bundler pipeline.

## Subdirectories And Files

- `github-data/` - Committed GitHub release/repo snapshots consumed by local release pages and project shell controls.
- `netbox-sdk-fixtures/` - Committed CLI/TUI fixtures generated from the local netbox-sdk checkout.
- `netbox-proxbox/` - Screenshot assets for the netbox-proxbox showcase page.
- `logos/` - Upstream NetBox logo assets used by the homepage architecture diagram.
- `llms.txt`, `robots.txt` - Public metadata files.

## Key Conventions

- Do not put secrets, private keys, tokens, or non-public data under `public/`.
- Snapshot/fixture files are generated by scripts but intentionally committed.
- Image filenames should stay stable once referenced by content. Rename to bust production caches when image contents change significantly.
- Add a scoped `CLAUDE.md` for any new non-trivial static data directory.
